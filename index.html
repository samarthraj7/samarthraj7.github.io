01

Summary Assist ‚Äì Enterprise Generative AI Platform

British Telecom | Multi-LLM RAG Pipeline | AWS & GCP Deployment

üéØ Situation: Need for Reliable Enterprise Summarization

Across large internal operations, teams produced massive volumes of service notes and technical documents. Manual summarization created delays and inconsistencies, while earlier automated solutions suffered from low factual accuracy and poor contextual grounding. Leadership aimed to design a unified Generative AI system that could deliver human-quality summaries at scale and reduce review overhead.

Challenge: Build a scalable, explainable, multi-model LLM workflow that integrates seamlessly into existing cloud infrastructure while maintaining accuracy, speed, and compliance.

üìã Task: Develop an End-to-End LLM Orchestration System

My Role: Lead Engineer ‚Äì Generative AI Integration & Evaluation

Objective: Design a configurable retrieval-augmented generation (RAG) platform that can orchestrate multiple large-language models, evaluate their outputs automatically, and scale across business domains via AWS and GCP.

Goal 1:

Cut summary rejection rate by more than half within the first release cycle.

Goal 2:

Standardize data pipelines and prompt orchestration for multiple LLM providers.

Goal 3:

Deliver a cloud-native, cost-optimized deployment serving thousands of internal users.

‚ö° Action: Engineering the Solution

1
Retrieval-Augmented Generation (RAG)
  • Implemented vector-based document retrieval using semantic embeddings to supply grounded context to LLMs and minimize hallucinations.
  • Designed dynamic context-window logic and adaptive prompt templates for varied document types.
2
Multi-Model LLM Integration
  • Connected multiple foundation models (GPT, Claude Sonnet, Llama, Gemini) through a config-driven orchestration layer.
  • Built automatic fallback and routing mechanisms based on latency, cost, and content complexity.
3
Evaluation & Feedback Framework
  • Created an automated scoring system inspired by RAGAs to measure relevance, factuality, and coherence.
  • Integrated human-feedback loops where reviewer corrections informed continuous prompt refinement.
4
Cloud Deployment & Scalability
  • Deployed services across AWS (Lambda, API Gateway, DynamoDB) and GCP (Vertex AI, BigQuery) for high availability.
  • Implemented load-balanced micro-services and asynchronous inference pipelines, improving response latency by ~50 % while reducing operating costs by ~30 %.
Problem-Solving Highlights
Challenge: Low factual accuracy in early summaries.
Solution: RAG grounding + automatic model selection to ensure contextual precision.
Challenge: Latency for long documents.
Solution: Parallel chunk retrieval and batched inference reduced end-to-end time by half.
Challenge: High inference cost at scale.
Solution: Adaptive routing to smaller or local models for lightweight requests.

The Winning Edge: Hybrid Content Based Recommender System (Cricket)

Published at WorldS4 London 2023, Springer

üéØ Situation: Strategy Meets AI

This project fused my cricket background with NLP to support split-second tactical decisions: shot selection, field placements, and bowling strategies.

The Problem: In-game decisions lean heavily on coach intuition and post-match analysis. We asked: can we surface real-time, AI-informed recommendations from 500+ historical scenarios?

üìã Task: Building an Intelligent Recommender

My Role: Lead NLP Engineer & System Architect

Team: 4 members (I led BERT/NLP and overall system design)

Goal 1:

Recommend alternate shots and field placements from 500+ scenarios

Goal 2:

Target 85%+ tactical alignment with expert analysts

Goal 3:

Process ball-by-ball commentary using NLP for actionable insights

Goal 4:

Context-aware outputs (same bowler, different situations ‚Üí different recs)

‚ö° Action: Building the Hybrid System

1
Dataset Creation: "StatHive"
  • Web Scraping: Ball-by-ball commentary (ESPNCricinfo, Cricbuzz)
  • Structured Data: Cricsheet JSON for match/ball details
  • BERT Keywords: Shot types, bowling styles, field cues from commentary
  • Feature Set: Line/length, player traits, runs/wickets/overs, venue/season
Innovation: Combined commentary NLP with structured stats.
2
The Hybrid Architecture: ANN + BERT
Component 1: ANN (Motive Prediction)
  • Inputs: Runs required, balls remaining, wickets down
  • Output: Attack (1) vs Defend (0) probability (sigmoid)
Component 2: BERT (Contextual Shot Suggestions)
  • Fine-tuned on cricket commentary
  • Uses bowler/batter, line/length, ground context to rank alternate shots
Integration:
  1. ANN infers motive from match state
  2. Filter to historical contexts matching the motive
  3. BERT recommends specific shots within filtered context
  4. System maps shots ‚Üí suggested field placements
4
Field Placement Logic
  • Template repository mapped from shot types to standard fields
  • Adjusted by bowler-batter combo, phase (PP/death), ground dimensions

85%

Tactical agreement with expert analysts (sampled scenarios)

500+

Historical scenarios analyzed

First-of-its-kind

BERT-driven cricket recommendations on commentary + stats

Applications
  • Pre-match practice plans (shot selection vs. bowlers)
  • In-game strategy cues (fields vs. high-probability shots)
  • Analyst tooling for opponent prep and weakness mapping
